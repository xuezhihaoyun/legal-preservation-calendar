# 财产保全到期提醒 Skill

上传财产保全文书 PDF，自动识别保全信息并按财产类型分别创建到期提醒日历。

## 功能

1. **PDF 识别** - 使用 MinerU OCR 提取 PDF 文字内容
2. **信息提取** - 自动提取：案号、被保全人、保全财产清单（银行账户/不动产/股权/车辆等）、各财产的保全期限
3. **智能计算** - 按不同财产的保全期限分别计算到期日，提前30天创建提醒
4. **日历创建** - 在苹果日历中批量创建提醒事件

## 触发词

- "保全提醒"
- "财产保全"
- "保全到期"
- "创建保全提醒"
- 直接发送 PDF 保全文书

## 支持文书类型

- 财产保全裁定书
- 保全结果告知书 / 保全结果通知书
- 协助执行通知书
- 财产保全情况告知书

## 识别字段

| 字段 | 说明 |
|------|------|
| 案号 | 标准案号格式，如：(2024)粤0305民初1234号 |
| 被保全人 | 被保全的当事人名称 |
| 财产类型 | 银行账户/不动产/股权/车辆/动产等 |
| 财产详情 | 账号、开户行、房产地址、股权数额等 |
| 保全期限 | 如：1年、2年、3年 |
| 起始日期 | 保全开始日期（如有） |

## 日历事件格式

- **标题**：`【保全到期】案号 - 财产类型（尾号/关键信息）`
- **提醒时间**：保全到期前30天
- **备注**：完整财产信息、保全期限、案号

## 混合保全处理

同一份文书可能包含不同保全期限的财产：
- 银行账户：通常保全1年
- 不动产：通常保全3年
- 股权：通常保全3年
- 车辆：通常保全2年

系统会自动识别各财产的期限，分别创建对应的提醒事件。

## 使用方法

1. 直接发送 PDF 文件给 AI
2. 或说："帮我创建保全提醒，文件在 /path/to/xxx.pdf"
3. AI 自动识别并创建日历事件
4. 查看日历确认提醒已创建

## 临时文件清理规则

**标准操作流程**：
- MinerU 转换生成的临时 MD 文件（位于原 PDF 同目录）**必须在日历事件创建完成后立即删除**
- 归档副本保留在 `~/.claude/skills/mineru-ocr/archive/` 目录，如需查阅可调取
- 仅删除临时工作文件，不清除归档和历史记录

## 注意事项

- 需要提前配置 MinerU API Token（参考 mineru-ocr skill）
- 如果 PDF 中未明确保全起始日期，默认以文书日期或当前日期推算
- 不同法院文书格式可能有差异，识别结果请人工核对
